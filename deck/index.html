The problem

Graded cards trade across a dozen venues. No two report the same price.

  • Thin markets: one wash trade can set the "price."
  • Every venue uses its own method — and has a stake in its own number.
  • No trustworthy price → no onchain RWA markets. Everything downstream is blocked.
The consequence

Today you take the number on faith.

Single-source, point-in-time, unauditable.

Architecture

Three tiers: source → consensus → chain

MemWal · the memory layer

Cold-start once. Smarter every run.

  • Price history: today's number is judged against the card's own past.
  • Reputation: sources that track consensus earn weight; drifters lose it.
  • Anomaly memory: past manipulation is pre-filtered next run.
  • One blob ID restores it all on a fresh machine — our production node does exactly this, every 6 hours.

Observable learning · the Walrus payoff

Not just stored on Walrus. Smarter because of it.

Trust, earned

Reliable sources count for more

Every source starts equal. The ones whose sales hold up earn weight (~88%); the noisy ones fade. Asking prices only set a ceiling — they never decide the final number.

Sharper over time

The price range tightens

Each round it remembers, the gap between the high and low estimate shrinks — so the number gets more confident.

Knows each card

Fewer false alarms

It learns what's normal for each specific card, so it stops flagging that card's real price swings.

Wipe the server and it rebuilds what it learned straight from Walrus. The memory outlives the machine.

Manipulation resistance

Thin markets are easy to spoof. We make it expensive and pointless.

At each source

Sanity checks

Ignore $0 glitches, flag a price that suddenly jumps 5×, distrust sales older than two weeks, and catch when one seller is behind too many of the sales.

Combining them

Throw out the outliers

Any sale that sits far from where the others cluster gets dropped before the price is set.

Combining them

Trust-weighted middle

The price is the middle of what's left, leaning on the most reliable, most recent sales. One fake trade can't move it.

Onchain

Agreement + challenge

Needs independent marketplaces that agree. Rare cards get a longer challenge window instead of being blocked. If sources clash, it holds off.

A faked sale is one voice against many real ones — it gets thrown out. Moving the price means faking several marketplaces at once.

Don't trust — verify

Every price ships with its receipts on Walrus.

A market cannot settle without its evidence blob. The contract aborts. Pull the blob, re-run the math — the price either checks out or it doesn't.

Most price feeds just hand you a number and ask you to trust it. You can’t see which sales went into it or what got thrown out, so if it’s wrong (or someone gamed it) you only find out after the money’s moved. Here a market can’t settle until the full working is published to Walrus and pinned onchain: the sales it used, the weights, everything it rejected. So if you doubt a price, you don’t take our word for it — you pull the file and re-run the math yourself. Walrus keeps the whole receipt cheaply and permanently; the chain just holds the one ID that proves nobody’s touched it.
